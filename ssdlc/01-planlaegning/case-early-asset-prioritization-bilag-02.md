# **Bilag 02 – Kickoff-skema til planlægningsmøde**

## **1. Formål**

Dette bilag indeholder et simpelt, praktisk anvendeligt kickoff-skema, der kan bruges af små udviklingsteams til at strukturere et 30–45 minutters sikkerhedsorienteret planlægningsmøde.

## **2. Indhold**

### **Kickoff-skema (version 1)**

**A. Formål med systemet (1–2 min)**

* Hvilket problem løser systemet?
* Hvem er primære brugere?

**B. Primære aktiver (5 min)**
Notér de 3–6 vigtigste aktiver:

| Aktiv | Hvorfor vigtigt? | Konsekvens ved kompromittering |
| ----- | ---------------- | ------------------------------ |
|       |                  |                                |
|       |                  |                                |
|       |                  |                                |

**C. Kritiske processer (5 min)**

* Hvilke funktioner må aldrig fejle?
* Hvilke ændringer ville være mest alvorlige?

**D. Foreløbige “må-ikke” handlinger (10 min)**
Definér 3–5 handlinger systemet aldrig må tillade:

| Må-ikke-handling | Begrundelse | Forretningspåvirkning |
| ---------------- | ----------- | --------------------- |
|                  |             |                       |
|                  |             |                       |
|                  |             |                       |

**E. Aftalt sikkerhedsniveau (5 min)**

* Hvilke sikkerhedskrav er absolut minimum?
* Skal teamet følge en “ASVS light”?
* Skal alle features have security review?

**F. Konkrete output (5 min)**
Efter mødet skal teamet have:

* En liste over aktiver
* En initial liste med må-ikke-handlinger
* En aftale om sikkerhedens serviceniveau
* Et sæt spørgsmål til næste fase (Analyse)

## **3. Refleksion**

Kickoff-skemaet blev testet af NordicApps på tre andre projekter. Den vigtigste erfaring er, at skemaet giver *retning* uden at skabe unødvendigt papirarbejde – og at selv et kort møde kan ændre hele teamets sikkerhedsfokus.


## **4. Referencer**

Ingen eksterne referencer – skemaet er et internt NordicApps-værktøj udviklet i FoU-projektet.
